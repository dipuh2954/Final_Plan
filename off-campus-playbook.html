DSA & Competitive Programming
90% of Indian tech companies test this before seeing your Android skills
This is the single most important parallel track to run alongside your Android learning. Almost every Indian product company — Swiggy, Flipkart, Razorpay, Zepto, CRED, PhonePe, Meesho, Dream11 — screens with 1–2 DSA rounds regardless of the role. Even "Android Developer" job descriptions will have a coding round. You cannot skip this. The good news: you don't need to be a competitive programming champion. You need to be LeetCode Medium-level reliable.
📚
Learn DSA in this exact order
Arrays → Strings → Hashing → Two Pointers → Sliding Window → Stack/Queue → Linked List → Binary Search → Recursion → Trees → Graphs → Dynamic Programming (basic). Don't jump to DP before mastering the earlier topics. Each topic: learn concept → solve 20 problems → move on.
🎯
Platform strategy — where to practice
Primary: LeetCode — do Blind 75 + NeetCode 150 as your core list. Secondary: GeeksForGeeks — great for Indian company-specific questions. Contests: CodeChef Starters & Codeforces Div.3 — participate weekly. Target: 200+ LeetCode problems solved (50 Easy, 120 Medium, 30 Hard) before applying.
⏱️
Daily commitment: 1.5–2 hours minimum
Solve 1–2 problems daily. On weekends, attempt a virtual contest. Do NOT stop doing DSA once you start Android — run them in parallel. Spend morning on DSA (fresh brain), evenings on Android development. This rhythm is used by every developer who landed their dream job from a tier-2/3 college.
📖
Resources to learn DSA (free & paid)
Free: Striver's A2Z DSA Sheet (best for Indian students), NeetCode.io, Take U Forward YouTube. Paid but worth it: Striver's DSA Course, AlgoMaster.io. Books: "Cracking the Coding Interview" by Gayle McDowell — read the relevant chapters before interviews.
🎓
MAKAUT Reality Companies like TCS, Infosys, Wipro use aptitude + coding tests heavily. But product companies (where salaries are ₹12–40 LPA) use pure DSA rounds. Your target should be product companies — and for those, Striver's sheet + LeetCode is your bible.

Academic Track
CGPA & Academic Standing
Don't let your degree become a gatekeeping filter against you
Many companies — especially off-campus applications — have automatic CGPA cutoffs of 6.5, 7.0, or even 7.5 in their application systems. Your resume never reaches a human if you're below the cutoff. You don't need to be a topper. You just need to clear the filter. Target: 7.5+ CGPA. Below 6.5 will close doors at many companies before you even start.
⚠️ Companies with CGPA cutoffs
  • Amazon: 7.0+ (varies)
  • Microsoft: 7.0+
  • Goldman Sachs: 7.5+
  • Flipkart: 6.5+
  • Google (India): 7.0+
  • Most MNCs via AMCAT/HackerRank
✅ Compensating for low CGPA
  • Strong projects override low CGPA at startups
  • Referrals can bypass ATS filters
  • GitHub activity shows real skills
  • Internship experience compensates
  • Some companies have no cutoff at all
  • Freelance work / own app on Play Store
💡
Practical Strategy If your CGPA is currently low (below 7.0), focus intensely on the next 2 semesters to bring it up. Simultaneously build a portfolio so strong that your GitHub and Play Store presence overrides the number. A 6.8 CGPA with 3 shipped Android apps beats a 9.0 CGPA with no projects at startups every single time.

Your Paper Identity
Resume & GitHub Profile
A recruiter spends 7 seconds on your resume. Make every second count.
Your resume and GitHub profile are the two documents that decide whether you get a call. From a MAKAUT college, your resume must lead with projects and skills, not education. Your GitHub must show real, active code — not a graveyard of tutorial repositories. These two documents need to tell one story: this person builds real things.
📝
Resume: Use Jake's Resume Template (LaTeX) or Overleaf
Use a single-page, ATS-friendly format. Section order for a fresher: Skills → Projects → Experience/Internships → Education. Never use tables, graphics, or columns — ATS (Applicant Tracking System) cannot parse them. Use Jake's Resume on GitHub or Overleaf's CS template. Font: plain, black text only.
🎯
Resume: How to write your Projects section
Each project: Name | Tech Stack | GitHub Link | Play Store Link (if published). Then 2–3 bullet points using the formula: "[Action verb] + [what you did] + [result/metric]". Example: "Built offline-first architecture with Room + Retrofit reducing API calls by 60%." Numbers matter. Even estimated numbers are better than none.
🔑
Resume: Use keywords that pass ATS
Job description says "Kotlin, Jetpack Compose, MVVM, REST APIs" — your resume must contain those exact words. Copy the job description keywords into your skills and project descriptions. This is how ATS works — it's keyword matching before human review. Tools: Jobscan.co to check your resume vs a JD.
🟢
GitHub: Green activity graph — non-negotiable
Commit code every single day. Even small things: fixing a bug, adding a test, improving a README. A GitHub profile with a full green graph is more convincing than 3 pages of resume. Pin your 6 best repositories. Each pinned repo needs: a clear README with screenshots, setup instructions, architecture explanation, and tech stack.
📸
GitHub: Professional profile setup
Create a GitHub Profile README (create a repo named after your username). Add: a short bio, tech stack badges, your current learning, contact links, and your pinned projects. Tools: github-readme-stats for dynamic stats cards. A polished GitHub profile gets forwarded by recruiters on LinkedIn.
❌ Resume Mistakes to Avoid
  • Objective statements ("Seeking opportunity...")
  • Listing skills you can't be interviewed on
  • Using "Familiar with" or "Exposure to"
  • Photos (illegal to ask in tech hiring)
  • More than 1 page as a fresher
  • Generic project descriptions with no metrics
  • Word .doc format (use PDF always)
✅ Resume Must-Haves
  • GitHub link + LinkedIn link + portfolio URL
  • Play Store link if app is published
  • Every tech skill you can be grilled on
  • Quantified project impact (metrics)
  • Certifications with dates
  • Hackathon wins if any
  • Open source contributions if any

Digital Presence
LinkedIn & Personal Brand
Recruiters come to you — but only if they can find you and you look worth calling
LinkedIn is not optional. It's where 80% of off-campus hiring happens in India through recruiter outreach, referrals, and direct applications. A poorly set-up LinkedIn profile is invisible. A well-optimized one gets you inbound recruiter messages even before you start applying. Building a personal brand (sharing your learning publicly) compounds this effect dramatically.
🖼️
LinkedIn Profile Optimization — The 7 must-dos
1. Professional photo (clear face, light background — not a selfie)
2. Headline: "Android Developer | Kotlin · Jetpack Compose · MVVM | B.Tech CSE @ BPPIMT" — not "Student at BPPIMT"
3. About section: 3–4 lines, first person, what you build + what you're learning + what you want
4. Featured section: Pin your best GitHub project and Play Store app
5. Skills: Add all relevant skills, get endorsements from classmates
6. Open to Work: Set it to "Recruiters only" once you're ready
7. Activity: Post at least 2x per week
📢
What to post on LinkedIn (this builds your brand)
Post about your learning journey — "I just learned how Hilt DI works, here's what clicked for me." Post project launches with screenshots. Post about bugs you fixed and what you learned. Post about what you're building. You don't need a big audience — just 3–5 recruiters seeing your posts is enough. Recruiters follow hashtags: #androiddev #kotlin #jetpackcompose #mobiledev
🌐
Build a Portfolio Website
A simple single-page portfolio at yourname.dev or yourname.github.io (free with GitHub Pages) adds immense credibility. Include: who you are, your projects with screenshots and links, your skills, and a contact form. Use a simple, fast template — don't over-engineer it. Having a personal domain (₹800/year on Namecheap) looks professional.
🔥
The Compounding Effect One LinkedIn post about a project you built can get 200–2000 views and 5–10 connection requests from developers and recruiters. Do this consistently for 6 months and you'll have an inbound pipeline of opportunities — people reaching out to YOU. This is the single most underused strategy by students from non-tier-1 colleges.

Experience Builder
Internship Strategy
One internship converts to 2x salary at placement — it's the biggest ROI investment you make
An internship — even a 2-month unpaid one at a startup — transforms your resume from "student with projects" to "professional with experience." It adds a company name under experience, gives you real production code exposure, and most importantly, gives you referrals. Most Indian product companies allow intern-to-full-time conversions. Internship hunting should start from Semester 3 onwards.
🔍
Where to find Android internships in India
LinkedIn Jobs — search "Android Developer Intern" + filter by India + Easy Apply
Internshala — largest Indian internship platform, many Android roles
Wellfound (AngelList) — startup internships, often higher quality work
Naukri.com — fresher/intern section
YCombinator job board — remote internships at YC startups
Direct cold email to startups — most effective method (covered in networking section)
📅
When to apply for internships
Start applying from Semester 3 (even if rejected — it teaches you the process). Seriously target from Semester 5. Apply to minimum 20 places per week — expect 5% response rate. Best timing: apply in Jan–March (summer internships) and Aug–Oct (winter internships). Quantity + consistency beats quality of applications at this stage.
💰
Don't reject unpaid or low-paid internships early on
A 2-month internship at a 3-person Kolkata startup with ₹5,000 stipend — where you ship real features to real users — is worth infinitely more on your resume than no internship. It gives you production experience, a reference, and talking points in every future interview. Take it if the work is real.
🌍
Remote internships — your biggest opportunity
Remote internships from US/EU companies pay $500–$2000/month even for students. Apply on: Wellfound, RemoteOK, WorkingNomads, YC job board. Requirements: strong English, good GitHub, Android skills. These opportunities exist and are actively seeking talented Indian developers. They're also incredible resume boosters.
⭐
GSOC & Competitive Programs Google Summer of Code (GSoC) — a paid 3-month open source program. ₹1.5–3 lakh stipend + Google on your resume. Applications open Jan–March. Start contributing to your target org from Semester 3. This alone can get you into Tier-1 companies that otherwise wouldn't look at MAKAUT resumes.
Also look at: MLH Fellowship, Outreachy, LFX Mentorship.

The Gauntlet
Interview Preparation
Getting shortlisted is 50% of the work. Clearing interviews is the other 50%.
Off-campus interviews at product companies typically have 4–6 rounds: Online Assessment (DSA) → Technical Round 1 (DSA live) → Technical Round 2 (Android deep dive) → System Design → HR/Offer. You need to prepare distinctly for each round. Failing one round ends the entire pipeline, so no round can be left unprepared.
💻
Round 1: Online Assessment (OA)
2–3 DSA problems in 90 minutes. Practice on HackerRank, HackerEarth, CodeSignal. Key habits: always test edge cases, write clean code with comments, don't optimize prematurely — a working O(n²) beats a broken O(n log n). Read problems twice before coding. Many students fail OAs by coding too fast.
🤝
Round 2: Live Coding (DSA with interviewer)
Think aloud — narrate your approach before writing code. Interviewers prefer candidates who communicate clearly over silent coders who get the right answer. Practice on Pramp.com, interviewing.io for mock interviews with real people. Structure: understand problem → brute force → optimize → code → test with examples.
📱
Round 3: Android Technical Deep-Dive
Be ready for: Kotlin concepts (coroutines, flow, sealed classes), Compose internals (recomposition triggers, stability), Architecture (why MVVM, what's Clean Architecture, how Hilt works), Memory management (what causes leaks, how to fix), Jetpack (Room, WorkManager, Paging), Performance (profiler, baseline profiles).
Most common question: "Walk me through how you'd architect [X feature]."
🏗️
Round 4: System Design (for mid+ roles)
For 0–1 YOE roles, this might be "Mobile System Design Light" — design the architecture of an app (e.g., design Instagram's feed). Study: "Mobile System Design" by Alex Lementuev (free on GitHub). Know: how to design offline-first apps, caching strategies, pagination, image loading, push notifications architecture.
🗣️
Round 5: HR & Behavioral
Use the STAR method (Situation, Task, Action, Result) for every behavioral question. Prepare answers for: "Tell me about yourself" (2 min, rehearsed), "Why this company?", "Biggest challenge you faced", "Where do you see yourself in 5 years?", "Why Android?". Research the company deeply — read their engineering blog, recent app updates, Glassdoor reviews.
💰
Salary Negotiation — don't skip this
Never give your expected salary first. Say: "I'd like to understand the full scope before discussing compensation." Research: Glassdoor, Levels.fyi, LinkedIn Salary, AmbitionBox for the exact company + role. Always negotiate — companies expect it. Even a ₹2 LPA increase is ₹20 LPA over 10 years. The worst they say is no.
Pramp.com — mock interviews interviewing.io Glassdoor — company research Levels.fyi — salary data AmbitionBox — India salaries Striver's SDE Sheet

Hidden Job Market
Networking & Cold Outreach
60–70% of jobs are never posted. They're filled through networks.
Networking is not about asking strangers for jobs. It's about building genuine relationships with people in the industry so that when an opening appears, your name comes up. From a non-IIT college, networking is your primary path to companies that would otherwise filter you at the CGPA/college name stage.
✉️
Cold LinkedIn DM / Email — the right way
Find: Android developers at companies you want to work at. Send a connection request with a note:
"Hi [Name], I'm a CSE student building Android apps in Kotlin/Compose. I've been following your work on [specific thing] and would love 15 minutes to ask about your experience at [Company]. Happy to share what I've been building too."
Be specific, be short, offer value, don't ask for a job. 10–20% will respond. That's enough.
👥
Communities to join immediately
GDG (Google Developer Groups) — find GDG Kolkata, attend meetups, volunteer to help organize. GDG connects you directly to Googlers and senior Android devs.
Android Discord servers (Kotlin, Android Developers)
Reddit: r/androiddev, r/cscareerquestions
Droidcon community
Local hackathons — Smart India Hackathon, HackWithInfy, Flipkart Grid
🎯
Referrals — the highest-value path into a company
A referral from an employee skips the ATS filter and goes directly to a hiring manager. To get referrals: connect with employees at target companies on LinkedIn → engage with their content for 2–3 weeks → then reach out asking if they'd be willing to refer you (most companies pay employees for successful referrals, so they're incentivized to help). Don't skip the relationship-building step.
💡
Hackathons — worth your time
Smart India Hackathon, Flipkart Grid, Amazon HackOn, HackWithInfy, DevFest. You don't need to win — participation shows initiative, and many sponsors offer internships/PPO to good teams. Hackathons also build your network fast since you spend 24–48 hours with motivated developers.
🏆
The Single Best Networking Move for a MAKAUT Student Join GDG Kolkata. Attend every event. Volunteer. Talk to every Android developer there. Within 6 months, you'll have more industry connections than most students from tier-1 colleges. GDG events are free and the quality of people you meet there is exceptional.

Credentials
Certifications
Not a replacement for skills — but they signal commitment and add legitimacy
Certifications don't get you jobs, but they add credibility when your college name doesn't. A Google-certified Android developer from BPPIMT is taken more seriously than an uncertified one. Prioritize certifications from Google, JetBrains, and Meta — these carry weight in Indian tech hiring.
🥇
Google Associate Android Developer (AAD) — Priority #1
The official Google Android certification. Tests real coding skills (you build features in a given app). Costs ~$149 (≈₹12,000). Recognized by most Indian companies. Valid for 3 years. Gives you a Google-signed digital badge you add to LinkedIn. Target this after completing Phase 3 of your Android roadmap. Worth every rupee.
🥈
Meta Android Developer Certificate (Coursera) — Priority #2
9-course Meta Professional Certificate on Coursera. Covers Android fundamentals through to app deployment. Costs ~₹1,500/month via Coursera subscription. More affordable than Google AAD. Great for early-stage (Phase 1–2) learning + certification. Recognized but less prestigious than Google AAD.
🥉
JetBrains Academy Kotlin Developer Certificate
Validates your Kotlin proficiency specifically. Useful since Kotlin is the language behind everything. JetBrains (the company that makes Kotlin and Android Studio) issuing a Kotlin certificate carries real weight. Available via JetBrains Academy.
☁️
Free Google/Android badges worth adding
Google Cloud Skills Boost — free badges for completing labs
Android developer.android.com codelabs — completion badges
Coursera: Android Basics with Compose (Google course, certificate available)
These are free or low-cost. Stack them on LinkedIn. They don't replace experience but fill the gap while you're building it.

Income While Studying
Freelancing & Side Income
Real client projects pay you AND build your portfolio simultaneously
Freelancing while studying serves three purposes at once: income (₹10,000–₹80,000/month as a student is very achievable), real-world project experience (better than personal projects for resume), and client references (a testimonial from a real business is gold). Start freelancing from Phase 2 onwards — you know enough to build simple apps.
🌐
Platforms to find freelance Android work
Upwork — best for long-term clients, high-quality work, but competitive. Build profile with first 2–3 jobs at lower rates to get reviews.
Fiverr — create gigs for specific Android tasks: "I will build an Android app with Kotlin and Compose"
Toptal — premium platform, requires screening, but pays $60–$150/hour
Freelancer.in — India-specific, lower rates but good for starting
PeoplePerHour — Europe-focused, decent rates
Local businesses (Kolkata/West Bengal) — most don't have apps yet. Visit shops, restaurants, local services. Offer to build an app for ₹15,000–₹50,000.
💡
What kind of Android work to offer
Beginner (Phase 2): Simple 3–5 screen apps (business card app, menu app, basic e-commerce UI) — charge ₹8,000–₹25,000
Intermediate (Phase 3): Apps with API integration, login, backend connectivity — charge ₹25,000–₹80,000
Advanced (Phase 5+): Full production apps with payment integration, push notifications, real-time features — charge ₹80,000–₹3,00,000+
Never undercharge for complex work — it attracts bad clients and devalues your skills.
📹
Content Creation — the long-term multiplier
Start a YouTube channel or blog about Android development. Even 1 video/week on something you learned. This seems slow — but developers with 5,000 YouTube subscribers get inbound job offers, paid sponsorships, course revenue, and freelance clients. Your content is a 24/7 portfolio. Channels like Philipp Lackner started as students. The compounding is real.
🚀
The Best First Client You'll Ever Have Look around your own city. Find a restaurant, gym, coaching centre, or local business with no app. Offer to build them a simple app for ₹10,000–₹20,000. Even if they negotiate you down to ₹8,000 — you now have a real client, real money, and a real deployed app. That one client often refers you to 3 more. This is how most successful freelance developers started.
