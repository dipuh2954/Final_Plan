Think through FCM flow
FCM flow: (1) User B sends a message. (2) Your server (or a Cloud Function) notices the new message. (3) Server sends FCM notification to User A's device token. (4) FCM delivers it to User A's device. For Semester 6, skip Cloud Functions — use a Firebase trigger or just send from the client when the app is in foreground.
